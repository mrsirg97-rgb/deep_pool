# DeepPool

**Pools that only get deeper.**

---

Every swap makes the pool stronger. 0.25% fee, 100% back to liquidity. No protocol extraction. No admin. No governance token. No LP farming. Just a pool that compounds.

---

## Why

Every DEX on Solana extracts from the pool. Raydium takes 16% of swap fees. Meteora takes 5-20%. Orca takes a cut. That value leaves the ecosystem and never comes back.

DeepPool takes nothing. The 0.25% swap fee stays in the pool as liquidity. Every trade deepens the pool. Every trade reduces slippage. Every trade makes the next trade better.

The pool is the product. There is nothing else.

## How It Works

DeepPool is a constant-product AMM (x * y = k). Each pool pairs a Token-2022 token with native SOL. No WSOL wrapping.

### Create

Anyone creates a pool by depositing SOL + tokens. They receive LP tokens representing their share. One pool per token mint.

### Swap

Traders swap SOL for tokens or tokens for SOL. The 0.25% fee is applied to the input before the constant product calculation. The fee stays in the pool — k increases after every swap.

### Add Liquidity

Anyone can add liquidity proportionally (SOL + tokens). They receive LP tokens representing their share of the growing pool.

### Remove Liquidity

LP holders burn their tokens to receive their proportional share of SOL + tokens. The pool is always redeemable.

That's it. Four instructions. No admin functions. No fee switches. No upgrades to pool state.

## The Math

Constant product: `x * y = k`

Swap output: `tokens_out = (effective_in * token_reserve) / (sol_reserve + effective_in)`

Fee: `effective_in = amount_in - (amount_in * 25 / 10000)`

After every swap: `k_new >= k_old`. Always. The fee guarantees k only grows.

Initial LP: `sqrt(sol * tokens) - MIN_LIQUIDITY`

The MIN_LIQUIDITY (1000 tokens) is permanently unrecoverable. This prevents the first-depositor rounding attack.

## Token-2022 Native

DeepPool works with Token-2022 tokens and native SOL. No WSOL. No wrapping. No unwrapping. No sync.

Token-2022 transfer fees (e.g. Torch's 0.04%) are handled by measuring net vault balances — the swap math uses what actually arrives, not what was sent.

## LP Tokens

LP tokens are Token-2022 mints with the pool PDA as mint authority. Nobody else can mint them. They represent a proportional share of the pool's reserves.

As fees compound into the pool, each LP token's underlying value increases. No staking. No claiming. No farming. Just hold.

## Self-Deepening

Every swap adds to k. Over thousands of trades:

- Slippage decreases
- Manipulation becomes more expensive
- Price discovery improves
- The pool becomes more useful

This is a flywheel with no extraction. Value generated by trading stays in the pool permanently.

## Designed for Torch

DeepPool replaces Raydium as the liquidity layer for Torch Market. After bonding, Torch migrates SOL + tokens into a DeepPool and burns its LP tokens. That liquidity is locked forever — the permanent base layer.

Community members can add liquidity on top. They hold LP tokens that appreciate as trading fees compound. Torch's depth-anchored risk model reads the pool's SOL balance directly — `pool_pda.lamports() - rent`. No adapter needed.

## What DeepPool Does Not Do

- Protocol fees
- Admin functions
- Governance
- Fee switches
- Concentrated liquidity
- Multiple fee tiers
- WSOL handling
- LP farming or staking rewards

## Security

- Pools are immutable after creation — no modification, no pause, no close
- Program is upgradeable — bug fixes don't affect existing pool state
- k is monotonically non-decreasing — formally verified
- All arithmetic is checked (u128 intermediaries, no overflow)
- Token-2022 transfer fees handled by net balance measurement
- First-depositor attack prevented by MIN_LIQUIDITY lock
- LP redemption uses floor division — always favors the pool
- 14 Kani formal verification proofs — all passing

## Verification

14 Kani proof harnesses. All passing. Covering:

- Fee conservation (fee + effective = input)
- Fee threshold correctness (0.25% exact)
- Swap output bounded (can't drain pool)
- k non-decreasing (core invariant)
- k strictly increases with fee (self-deepening property)
- Swap monotonicity (more in = more out)
- LP sqrt correctness (initial mint)
- LP proportional minting
- LP redemption bounded (can't take more than exists)
- LP full redemption (100% LP = 100% reserve)
- LP redemption monotonicity
- Sell-side output bounded

## Constants

| Parameter | Value |
|-----------|-------|
| Swap fee | 0.25% (25 bps) |
| Protocol fee | 0% |
| Min initial SOL | 0.1 SOL |
| Min initial tokens | 1 token (6 decimals) |
| MIN_LIQUIDITY | 1000 (locked on first deposit) |
| LP decimals | 6 |
| Pool per mint | 1 (PDA enforced) |
